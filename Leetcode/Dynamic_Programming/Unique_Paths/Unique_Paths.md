### 1. 核心題意與挑戰

機器人位於一個 `m x n` 網格的左上角（起始點設為 `[0, 0]`）。
機器人每次只能 **向下** 或 **向右** 移動一步。機器人試圖達到網格的右下角（目標點 `[m-1, n-1]`）。
問總共有多少條不同的路徑？

* **隱藏挑戰**：不可用純遞迴 (DFS) 去窮舉，因為樹的展開太過龐大（組合爆炸），一定會 Timeout。

---

### 2. 解法對比與完整程式碼

#### A. 二維動態規劃 (2D DP) —— **經典網格 DP**

**思路**：
定義 `dp[i][j]` 為到達座標 `(i, j)` 的路徑數量。
因為機器人只能往右或往下走，所以到達 `(i, j)` 的路徑，必定只能從其 **左邊** `(i, j-1)` 或 **上面** `(i-1, j)` 走來。
狀態轉移：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`。
邊界條件：第一列和第一行因為各自只有一條直線路徑可達，所以值皆為 `1`。

* **優點**：非常直觀，圖形化解釋容易。
* **缺點**：空間複雜度為 $O(m \times n)$。

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 建立 m x n 矩陣並全部初始化為 1
        # 因為第一欄跟第一列的值本來就該是 1，這樣省去後續的處理
        dp = [[1] * n for _ in range(m)]
        
        # 從 1 開始遍歷，直接進行相加
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = dp[i-1][j] + dp[i][j-1]
                
        return dp[m-1][n-1]
```

#### B. 一維動態規劃 (Space-Optimized DP) —— **降階最佳化**

**思路**：
觀察方程式 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`。
當我們在計算新的一列時，只需要用到「上方元素 (`dp[j]`，尚未更新)」與「左方元素 (`dp[j-1]`，已經更新)」。
所以只需要一條長度為 `n` 的陣列就可以一列一列往下「刷」過去。

* **時間複雜度**：$O(m \times n)$
* **空間複雜度**：$O(n)$

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 初始化第一列皆為 1
        dp = [1] * n
        
        # 一列一列往下壓
        for i in range(1, m):
            for j in range(1, n):
                # dp[j] (本身，即上方舊值) + dp[j-1] (左邊剛算出的新值)
                dp[j] = dp[j] + dp[j-1]
                
        return dp[-1]
```

#### C. 數學排列組合 (Combinatorics) —— **$O(1)$ 空間數學解**

**思路**：
不論你選哪條路，走到終點總共要往下走 `m-1` 步，往右走 `n-1` 步。總共走 `(m-1) + (n-1)` 步。
題目轉化為：在總步數中，挑選 `m-1` 步往下的組合數有多少？
即組合數學中的：$C_{m-1}^{(m-1)+(n-1)}$

```python
import math

class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # math.comb(n, k) 也就是 C(n, k)
        return math.comb(m + n - 2, m - 1)
```

### 3. 實務應用場景

#### 1. 影像處理與最短路徑找尋 (Image Processing / Seam Carving)
* **應用**：影像處理中縮小圖片的 Seam Carving 演算法就是大量利用類似的 Grid DP 計算圖片上一群像素走向的能量最低路徑。

#### 2. 控制理論中的狀態轉移矩陣 (State Transition Matrix)
* **應用**：用於計算在馬可夫決策過程中，由初始狀態到達目標狀態的方法數或機率期望值歸一化。

### 4. 總結筆記

| 比較維度 | 2D DP | 1D DP | 數學公式法 |
| --- | --- | --- | --- |
| **時間複雜度** | $O(m \times n)$ | $O(m \times n)$ | $O(\min(m, n))$ (階乘計算) |
| **空間複雜度** | $O(m \times n)$ | $O(n)$ | $O(1)$ |
| **面試期待** | 必須說明清楚轉移過程 | 至少寫出這個版本才算過關 | 可當作炫技或閒聊，不建議當主解法 |
