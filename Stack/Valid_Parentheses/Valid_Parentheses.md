### 1. 核心題意與挑戰

給定一個只包括 `'('`, `')'`, `'{'`, `'}'`, `'['`, `']'` 的字串 `s` ，判斷字串是否有效。
有效字串需滿足：
1. 左括號必須用相同類型的右括號閉合。
2. 左括號必須以正確的順序閉合。
3. 每個右括號都有一個對應的相同類型的左括號。

* **隱藏挑戰**：
  * LeetCode 最經典的第一道 Stack 題。這題如果寫得醜，面試官對你的印象會大打折扣。
  * 需要處理各種邊界：例如一開始就是右括號 `]`，或是最後左括號太多 `((()`。

---

### 2. 解法對比與完整程式碼

#### 唯一正解：堆疊 (Stack) + 雜湊表 (HashMap) —— **最乾淨的映射法**

**思路**：
遇到左括號，我們就把它「推入 (Push)」堆疊。
遇到右括號，我們就檢查堆疊的頂端「彈出 (Pop)」是否是一對的左括號。
如果不是一對，或者堆疊根本就是空的，那就直接回傳 `False`。
掃瞄完全部字串後，如果堆疊內還有剩東西（代表左括號太多），也是 `False`。

* **進階技巧**：
  為了不要寫大量的 `if...elif...elif...`，我們可以用一個 `HashMap` 把 右括號 與 左括號 的對應關係存起來。

* **時間複雜度**：$O(N)$
* **空間複雜度**：$O(N)$ (最差情況下全是左括號，Stack 裝滿)

```python
class Solution:
    def isValid(self, s: str) -> bool:
        # 面試官最愛看的 mapping 寫法，擴展性強
        bracket_map = {
            ')': '(',
            '}': '{',
            ']': '['
        }
        stack = []
        
        for char in s:
            if char in bracket_map:
                # 遇到右括號了
                # 拿取 stack 最頂端的元素。如果 stack 是空的，給個隨便的假字元 '#'
                top_element = stack.pop() if stack else '#'
                
                # 如果彈出來的不是配對的左括號，直接失敗
                if bracket_map[char] != top_element:
                    return False
            else:
                # 遇到左括號，推入 stack
                stack.append(char)
                
        # 最後如果 stack 是空的代表全配對成功，否則就是有殘留的左括號
        return len(stack) == 0
```

### 3. 實務應用場景

#### 1. 編譯器與 IDE 的語法高亮與錯誤偵測 (Syntax Checking)
* **應用**：你現在用的 VSCode 當你不小心少打一個括號時立刻報紅字，底層的引擎之一就是這套 Stack 演算法。

#### 2. DOM Tree 的 XML/HTML 標籤解析 (HTML Parsing)
* **應用**：瀏覽器在解析網頁時，`<div>` 就是左括號，`</div>` 就是右括號。如何確保 DOM 樹節點被完美嵌套而不交錯，就是這套邏輯的字串加強版。

### 4. 總結筆記

| 面試加分清單 | 解說 |
| --- | --- |
| **用 Dictionary 替代 if-else** | 這是向面試官展示「工程維護度」的最佳時機。如果未來要新增一種角括號 `< >`，用字典只要加一行，用 if-else 就要多寫好幾行。 |
| **`pop() if stack else '#'`** | 處理防呆。當遇到字串字首就是右括號 `]` 時，如果直接 `pop()` 會引發 `IndexError`，這是一種非常優雅的邊界處理守門員寫法。 |
| **`return not stack`** | 最後的檢查很容易被忘記。例如 `s = "["`，如果不檢查直接 return `True`，面試就會直接陣亡。 |
