### 1. 核心題意與挑戰

和前一題的條件一模一樣，給你一個被旋轉過的升序陣列 `nums`。
但這次不是找最小值，而是要給定一個目標值 `target`，如果 `target` 存在於陣列中，回傳它的索引；如果不存在，回傳 `-1`。

* **關鍵限制**：
  * **必須**在 $O(\log N)$ 時間複雜度內解決。
* **隱藏挑戰**：
  * 面試中，這題比上一題更常被考。因為它不只要找斷點，還要判斷 `target` 到底落在哪半邊。邏輯的分支會變得非常複雜。

---

### 2. 解法對比與完整程式碼

#### 唯一正解：進階變形二元搜尋 (Advanced Binary Search)

**思路**：
這題是「找出那一半是連續有序的」的哲理延伸。
每次把陣列從 `mid` 切開，**一定有一半是完美有序的**。
太棒了！只要有一半是有序的，我們就可以輕易判斷 `target` 放不放得進這個有序區間裡。

1. **找出有序半邊**：
   * 如果 `nums[left] <= nums[mid]`，代表「左半邊」是完美有序的。
   * 否則，必定是「右半邊」完美有序。
2. **審問 target**：
   * 情況 A (左半有序)：問 `target` 是不是舒舒服服地躺在 `[nums[left], nums[mid]]` 的範圍內？
     * 如果是，那我們就把搜索範圍縮小到左邊 (`right = mid - 1`)。
     * 如果不是，那它一定跑到右邊那坨亂七八糟的裡面了 (`left = mid + 1`)。
   * 情況 B (右半有序)：問 `target` 是不是舒舒服服地躺在 `[nums[mid], nums[right]]` 的範圍內？
     * 如果是，縮小範圍到右邊 (`left = mid + 1`)。
     * 如果不是，那就是跑到左邊去了 (`right = mid - 1`)。

* **時間複雜度**：$O(\log N)$
* **空間複雜度**：$O(1)$

```python
from typing import List

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            
            # 幸運地直接中獎
            if nums[mid] == target:
                return mid
                
            # 1. 左半段是完美有序的
            # 這裡必須要有等號，因為當 left == mid 時，依然算作有序
            if nums[left] <= nums[mid]:
                # 檢查 target 是否正好落在這段完美有序的區間內
                if nums[left] <= target < nums[mid]:
                    right = mid - 1 # 是的話，往左找
                else:
                    left = mid + 1  # 不是的話，只能去右邊的斷層區找了
                    
            # 2. 右半段是完美有序的
            else:
                # 檢查 target 是否正好落在這段完美有序的區間內
                if nums[mid] < target <= nums[right]:
                    left = mid + 1  # 是的話，往右找
                else:
                    right = mid - 1 # 不是的話，只能去左邊的斷層區找了
                    
        return -1
```

### 3. 實務應用場景

#### 1. 資料庫的 B+ Tree 分散式索引損壞修復
* **應用**：在分散式資料庫中，有時候一個順序的 Range Partition 發生節點飄移，導致索引樹底層的連續鏈結串列變成了旋轉陣列。透過這個 BS 的修改版，可以在不重建整個 Partition 的情況下，依然維持 $O(\log N)$ 的高速查詢，直到系統排程的離線資料重整完成。

### 4. 總結筆記

| 面試生死線 | 解說 |
| --- | --- |
| **`nums[left] <= nums[mid]` 的等號** | 極度重要！當陣列剩下兩個元素例如 `[3, 1]` 時，`mid` 會算成 0，此時 `nums[left]` 等於 `nums[mid]` 就是 3。如果沒加等號，程式會誤以為右半邊是有序的，從而判斷錯誤。在長度極短時這是一個防呆機制。 |
| **先判斷有序，再判斷邊界** | 千萬不要一開始就想去猜 `target` 在哪，因為陣列有斷層。**永遠先找出哪一半是正常的**，因為正常的東西才有規律可用（最大值最小值一目了然）。這是一個深刻的人生哲理。 |
