### 1. 核心題意與挑戰

給你一個整數陣列 `nums`，請回傳一個陣列 `answer`，其中 `answer[i]` 等於 `nums` 中除了 `nums[i]` 之外其餘各元素的乘積。

* **關鍵限制**：
  * 保證陣列中所有數字相乘的結果在 32 位元整數範圍內。
  * **禁止使用除法 (Division)**。
  * 必須在 $O(N)$ 時間內完成。
* **面試官超時限挑戰 (Follow-up)**：你能用 $O(1)$ 的額外空間複雜度來完成嗎？（輸出陣列不計入空間複雜度）。

---

### 2. 解法對比與完整程式碼

#### A. 左右前綴乘積陣列法 (Prefix / Suffix arrays)

**思路**：
既然不能用除法（把全部乘起來再除掉自己），那就把乘積拆兩半：
對於元素 `nums[i]` 來說，除自己以外的乘積 = **(所有的左邊元素的乘積) $\times$ (所有的右邊元素的乘積)**。
我們可以先掃描一次，建立一個計算左邊前綴乘積的陣列 `L`。
再反向掃描一次，建立一個計算右邊後綴乘積的陣列 `R`。
最後將兩者對位相乘即可。

* **時間複雜度**：$O(N)$
* **空間複雜度**：$O(N)$，因為創建了左陣列和右陣列。未達到面試官最滿意的條件。

#### B. 壓縮空間的常數解 (Space-Optimized O(1)) —— **強烈推薦唯一解法**

**思路**：
同上，我們依舊利用左乘右的概念，但我們**不創建**額外的空間陣列！
我們直接把 `answer` 陣列當作先前的 `L` 陣列來用，先從左到右掃描算出所有的左半部乘積。
然後，我們再從右到左掃描，這次不用新建陣列，而是維護一個動態的常數變數 `right_prod`，邊算邊乘上 `answer` 原本儲存的左半部結果，即時更新。

```python
from typing import List

class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        # 用來作為輸出的陣列 (不計入空間複雜度)
        answer = [1] * n
        
        # 1. 從左往右掃描，計算每個元素「左邊的總乘積」
        # 第一個元素左邊沒有東西，所以初始化為 1
        left_prod = 1
        for i in range(n):
            answer[i] = left_prod
            left_prod *= nums[i]  # 更新給下一個元素用
            
        # 此時 answer 陣列裡裝的是：[1, nums[0], nums[0]*nums[1], ...]
        
        # 2. 從右往左掃描，計算每個元素「右邊的總乘積」並馬上乘進 answer
        # 最後一個元素右邊沒有東西，所以初始化為 1
        right_prod = 1
        for i in range(n - 1, -1, -1):
            # 原本的 answer[i] 是左邊總乘積，乘上右邊總乘積就是答案
            answer[i] *= right_prod
            right_prod *= nums[i] # 更新給下一個(其實是左邊那個)元素用
            
        return answer
```

### 3. 實務應用場景

#### 1. 地理空間座標/訊號變換的權重屏蔽 (Masking & Signal Processing)
* **應用**：在處理多維度感測器訊號（例如有 4 個鏡頭的自駕車），若要計算「去除第 i 個鏡頭訊號」時系統預期收到的底噪干擾倍率，就會需要這類排除單一對象的前推/後驗乘積模型。

#### 2. SQL / MapReduce 的窗函數優化 (Window Functions without Division)
* **應用**：有時候在分散式純量計算中，資料包含 `0` 會導致無法使用反向除法 (Division by Zero)。Left/Right Prefix 的拆解演算法便可以完美避開 `0` 的崩潰問題並平行計算。

### 4. 總結筆記

| 面試加分點 | 解釋 |
| --- | --- |
| **不用除法** | 就是因為有 `0` 的存在。如果可以除，遇到一個 `0` 整盤皆崩，遇到兩個 `0` 時答案全為 `0` 的處理極為繁瑣。 |
| **$O(1)$ 空間優化** | 將原本應當分開存放的左側狀態，預加載進 `answer` 陣列中。這個「**將結果陣列拿來當作中繼變數使用**」的技巧非常常見！ |
