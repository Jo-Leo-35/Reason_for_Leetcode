### 1. 核心題意與挑戰

這題是 **House Robber** 的進階版。
所有的房屋都圍成一圈，這意味著**第一間房屋和最後一間房屋是緊挨著的**。
你不能在同一個晚上闖入相鄰的兩間房屋。請計算你在不觸動警報裝置的情況下，能夠偷竊到的最高金額。

* **關鍵限制**：陣列成環 (Circular Array)，牽一髮而動全身，首尾相連。
* **隱藏挑戰**：如何打破環狀依賴？如果你偷了第一家，就絕對不能偷最後一家；如果沒偷第一家，就可以偷最後一家。

---

### 2. 解法對比與完整程式碼

#### A. 破解環狀陣列：拆分成兩個線性 DP —— **推薦面試解法**

**思路**：
既然首尾互斥，那只能有兩種大方向的可能：
1. **考慮偷第一間房到倒數第二間房**（即索引 `0` 到 `n-2`）。不含最後一間。
2. **考慮偷第二間房到最後一間房**（即索引 `1` 到 `n-1`）。不含第一間。

對於這兩種情況，我們都可以直接套用 **House Robber I** 的線性 DP 演算法，最後再取這兩種情況的最大值即可。

* **優點**：邏輯清晰，完美重用已經寫過的 House Robber 各模塊。
* **空間複雜度**：透過傳入子陣列（或利用指針），結合空間最佳化，可達到 $O(1)$ 空間。

```python
from typing import List

class Solution:
    def rob(self, nums: List[int]) -> int:
        if len(nums) == 0:
            return 0
        if len(nums) == 1:
            return nums[0]
            
        # 將 House Robber I 包裝成輔助函數
        def helper(houses):
            rob1, rob2 = 0, 0
            for money in houses:
                temp = max(money + rob1, rob2)
                rob1 = rob2
                rob2 = temp
            return rob2
        
        # 情況一：不包含最後一家 nums[:-1]
        # 情況二：不包含第一家 nums[1:]
        return max(helper(nums[:-1]), helper(nums[1:]))
```

*註：在面試中，可以直接重用 `helper` 函數的寫法，對於模組化程式碼會是非常好的展現。*

### 3. 實務應用場景

本題帶來最重要的工業界觀念：**「打破循環依賴 (Breaking Circular Dependencies)」**。

#### 1. 環狀網路拓樸 (Ring Network Topology)
* **應用**：環狀網路中的節點負載平衡或更新。當不允許相鄰節點同時離線升級時，如何規劃最高效的全局升級路線？透過在任意處切斷成線狀來評估。

#### 2. 定期輪值班表的衝突避免 (Scheduling)
* **應用**：輪值表跨月、跨年的連續性排版。月底跟下個月初會形成循環相鄰，若有「做一不能休一」的勞基法限制，通常就是透過拆解跨月的狀態，當作線性問題求解再聚合。

### 4. 總結筆記

| 比較維度 | 拆分線性 DP |
| --- | --- |
| **時間複雜度** | $O(N)$ (走訪兩次子陣列) |
| **空間複雜度** | $O(N)$ (若有切片 Slice) / $O(1)$ (若用 Pointer) |
| **代碼重用性** | 極高，直接調用 House Robber I 邏輯 |
| **面試核心** | 「取頭去尾」與「取尾去頭」的解耦合思想 |
