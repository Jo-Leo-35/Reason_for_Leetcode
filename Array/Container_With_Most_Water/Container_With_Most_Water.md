### 1. 核心題意與挑戰

給定一個長度為 `n` 的整數陣列 `height`。有 `n` 條垂直線，第 `i` 條線的兩個端點是 `(i, 0)` 和 `(i, height[i])`。
找出其中的兩條線，使得它們與 x 軸共同構成的容器可以容納**最多**的水。

* **隱藏挑戰**：
  * 面試常見的第一道雙指針題。
  * $O(N^2)$ 的暴力組合法一定會超時。
  * 容量公式為：`min(height[left], height[right]) * (right - left)`

---

### 2. 解法對比與完整程式碼

#### A. 雙向夾擊法 (Two Pointers) —— **最完美的 O(N) 貪婪解**

**思路**：
我們將兩個指針 `left` 和 `right` 分別放置在陣列的最左邊和最右邊。
在這個最寬的狀態下，計算容量並記錄下來。
**決定誰移動？** 這水桶的短板決定水位。我們只能嘗試**移動較短的那邊**！
* 如果移動較高的那邊，寬度必定減少，而且高度最多也只能維持現狀（因為還是被短板限制著），容量絕對不可能變得更大。
* 如果移動較短的那邊，雖然寬度減少了，但我們**有機會**遇到一根非常高的柱子，從而彌補寬度損失，得到更大的容量。

* **優點**：邏輯極為簡單暴力。

```python
from typing import List

class Solution:
    def maxArea(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        max_water = 0
        
        while left < right:
            # 計算當前容量 Width * Height
            current_water = (right - left) * min(height[left], height[right])
            # 更新歷史最大值
            max_water = max(max_water, current_water)
            
            # 將比較短的那一根柱子往內移動，尋找可能更高的新柱子
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
                
        return max_water
```

### 3. 實務應用場景

#### 1. 排程時間與算力調配的 Trade-off
* **應用**：有這種類似「寬度與短板相乘求極大」的模型，例如在計算多節點叢集的運算效能時，系統能支撐的最大吞吐量往往由叢集中「效能最差的機器」乘上「機器平行數」來決定。透過類似的貪婪排除法，可以幫助捨棄爛機器並換取更高集中的算力。

#### 2. 版面設計的彈性縮放排列 (UI Layout Aspect Ratio)
* **應用**：在開發前端 Flexbox 或是響應式設計時，圖片容器的最大化有時候就是基於邊框容器限制下的最大顯示比例探討。

### 4. 總結筆記

| 解題心法 | 說明 |
| --- | --- |
| **短板效應** | 容器的水位高低永遠由比較短的那根柱子決定。 |
| **捨棄最劣解** | 因為移動比較長的那根柱子「百害而無一利」 (寬度變窄、短板依舊存在)，所以我們只移動短板。這個貪婪策略保證了我們不會漏掉真正的最佳解。 |
| **時間複雜度** | $O(N)$，兩個指針相向而行，走訪完整個陣列。 |
| **空間複雜度** | $O(1)$，不需要額外陣列。 |
