### 1. 核心題意與挑戰

給定一個未排序的整數陣列 `nums`，找出最長連續元素序列的長度。
（連續序列指的是數字依照 `1, 2, 3...` 排列，但不要求它們在原陣列中的物理位置是連續的）。

* **關鍵限制**：
  * 你必須設計並實現時間複雜度為 **$O(N)$** 的演算法解決此問題。
* **隱藏挑戰**：
  * 看到「找連續」，第一反應都是排序 ($O(N \log N)$)，但這題明令禁止。如何在未排序的情況下用 $O(N)$ 找到連續段的頭尾？答案必然與 `Hash` 相關。

---

### 2. 解法對比與完整程式碼

#### 唯一正解 (HashSet 找源頭法) —— **思維極度簡潔的高雅解**

**思路**：
要把這題用 $O(N)$ 解出來，我們只需關注兩件事：
1. **去重並能 $O(1)$ 查找**：先把所有數字丟進一個 `HashSet` 裡面。
2. **只從「連續序列的起點」開始往上數**：如果我們拿到一個數字 `x`，只要確認 `x - 1` **不存在**於 Set 中，那這個 `x` 必定是某段連續序列的老大（最底部的源頭）。
3. 從這個源頭不斷往上查找 `x + 1`, `x + 2`... 直到找不到為止，這就是一段完整的序列長度。

* **為什麼這是 $O(N)$**？
  雖然裡面包了一個 `while` 迴圈不斷往上數 `curr_num += 1`，但仔細觀察：
  **只有真正的「源頭節點」才會進入這個 while 迴圈**！
  如果一個連續序列長度為 $L$，我們只會在頭節點花 $O(L)$ 時間掃描，而不會在中間節點重複掃描。因此陣列中每個節點都「最多」只會被掃過兩次（一次外層判定，一次內層延伸），總時間複雜度嚴格保證是 $O(N)$。

* **優點**：邏輯漂亮，完全符合 $O(N)$ 的硬性要求。

```python
from typing import List

class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 去重並支援 O(1) 尋找
        num_set = set(nums)
        longest = 0
        
        for num in num_set:
            # 判斷是否為「序列的頭」
            # 只要 num-1 不在 set 裡，代表沒有人排在它前面，它就是最源頭的頭！
            if (num - 1) not in num_set:
                curr_num = num
                curr_streak = 1
                
                # 從這個頭開始，拼命往上尋找連續的數字
                while (curr_num + 1) in num_set:
                    curr_num += 1
                    curr_streak += 1
                    
                # 紀錄最長的一段
                longest = max(longest, curr_streak)
                
        return longest
```

### 3. 實務應用場景

本題體現了「Union-Find (併查集)」退化為單鏈時的終極奧義，但在實務上更多用於：

#### 1. 資料庫的碎片重組 (Defragmentation logic)
* **應用**：在記憶體管理或硬碟區塊管理中，給定一堆零碎的、亂序的 Block ID。系統要判斷哪一段具有最長且能合併的連續物理可用空間。利用 `HashSet` 找尋 `Block_ID - 1` 不存在的源頭，正是 $O(N)$ 合併碎片的基礎。

#### 2. 分散式系統的訊息重組 (UDP Sequence Reconstruction)
* **應用**：UDP 封包是沒有順序保障的。當收到一堆帶有 Sequence Number 的封包亂序抵達時，要在最短時間內拼湊出目前「最長已經連續接收到的封包長度」，可以用 Set 去掉重複接收的干擾，並利用找源頭法高速推演。

### 4. 總結筆記

| 解題心法 | 說明 |
| --- | --- |
| **不要排序！** | 看到 $O(N)$ 限制，直接放棄 `.sort()`。 |
| **使用 HashSet** | `$O(1)$` 查找是此題的心臟。 |
| **`if (num - 1) not in set:`** | 這是整題的靈魂！**主動濾除非頭節點**，保證了不管陣列多長，不會重複計算同一條連續鏈結，從而保證 $O(N)$。 |
